## 一、Baostock

### 基本信息

- **官网**: http://baostock.com
- **安装**: `pip install baostock`
- **账号**: 无需注册，免费使用
- **平台**: Mac / Windows / Linux 全兼容（纯 Python）
- **数据延迟**: T+1（当天收盘后次日可获取当日数据）

### 能获取的数据

| 数据类型 | 接口 | 说明 |
|---|---|---|
| 日线 K 线 | `query_history_k_data_plus(frequency='d')` | 开高低收、成交量、成交额、换手率、涨跌幅 |
| 5 分钟 K 线 | `query_history_k_data_plus(frequency='5')` | 同上，5分钟粒度 |
| 15/30/60 分钟 K 线 | `query_history_k_data_plus(frequency='15/30/60')` | 同上 |
| 全 A 股列表 | `query_stock_basic()` | 代码、名称、上市状态、股票类型 |
| 行业分类 | `query_stock_industry()` | 证监会行业分类（83个大类） |
| 沪深300成分股 | `query_hs300_stocks()` | 指定日期的成分股列表 |
| 中证500成分股 | `query_zz500_stocks()` | 同上 |

### 不能获取的数据

- 实时行情（只有历史数据）
- 集合竞价逐笔明细
- 概念板块分类
- Level-2 数据
- 1 分钟 K 线（接口存在但常返回空）

### 本项目中的使用方式

**1. 历史数据下载（run_vnpy.py --download）**

通过 `vnpy_baostock` 适配器将数据存入 VnPy 的 SQLite 数据库，供回测引擎使用。

**2. 每日信号生成（daily_signal.py）**

每天收盘后下载最新日线，用回测引擎跑策略到当天，输出买卖信号。

**3. 全市场连板计算（scan_v2.py）**

遍历全 A 股 5199 只股票的一年日线，逐只计算每天的连续涨停天数，找出所有 N 晋 N+1 的事件。判断涨停的标准：主板涨幅 >= 9.8%，创业板/科创板涨幅 >= 19.5%。

**4. 竞价换手率计算（scan_v2.py）**

用 5 分钟 K 线的第一根（9:30-9:35）成交量作为竞价成交量的近似值。流通股本通过 `成交量 / 换手率 * 100` 从日线数据反算。竞价换手率 = 竞价成交量 / 流通股本 * 100。

**5. 高开幅度计算**

日线数据中 `(开盘价 - 昨收) / 昨收 * 100`。

### 使用示例

```
登录:       bs.login()
查日线:     bs.query_history_k_data_plus('sh.600519', 'date,open,close,turn,pctChg', start_date, end_date, frequency='d')
查5分钟线:  bs.query_history_k_data_plus('sh.600519', 'date,time,volume', start_date, end_date, frequency='5')
查行业:     bs.query_stock_industry()
查股票列表:  bs.query_stock_basic()
登出:       bs.logout()
```

### 注意事项

- 每次使用前需 `bs.login()`，用完 `bs.logout()`
- 复权方式：`adjustflag='3'`（不复权）用于实际价格判断涨停，`adjustflag='2'`（前复权）用于回测
- 5 分钟 K 线数据只保留最近几个月，太早的会返回空
- 批量查询 5000+ 只股票时，单次约 12 分钟完成

## 二、easyquotation（新浪行情）

### 基本信息

- **GitHub**: https://github.com/shidenggui/easyquotation （5.1k Star）
- **安装**: `pip install easyquotation`
- **账号**: 无需注册
- **平台**: Mac / Windows / Linux 全兼容（纯 Python）
- **数据延迟**: 约 3-5 秒（HTTP 轮询）

### 能获取的数据

| 数据类型 | 说明 |
|---|---|
| 实时价格 | 最新价、开盘价、最高价、最低价、昨收价 |
| 买卖五档 | bid1-bid5（价格+数量）、ask1-ask5（价格+数量） |
| 成交数据 | 成交量（股）、成交额（元） |
| 时间 | 日期、时间 |
| 股票名称 | 中文名 |
| 全市场代码列表 | `get_stock_codes()` 返回全 A 股代码 |

### 不能获取的数据

- 历史 K 线
- 分钟线
- 财务数据
- 概念/行业分类
- 盘后数据不更新

### 本项目中的使用方式

**1. SinaGateway（sina_gateway.py）**

自研的 VnPy Gateway 适配器，将 easyquotation 的数据格式转换为 VnPy 的 TickData 对象，推送给 VnPy 引擎。

工作方式：
- 连接时调用 `get_stock_codes()` 获取全市场 5610 只股票代码，注册为 VnPy 合约
- 启动一个后台线程，每 3 秒调用 `quotation.real(subscribed_codes)` 获取实时行情
- 将返回的字典转换为 TickData，通过 `on_tick()` 推送给引擎
- PaperAccount 接收到 TickData 后进行仿真撮合

**2. 实时行情查看**

直接在命令行查看个股实时数据，用于盘中监控。

### 使用示例

```
初始化:     quotation = easyquotation.use('sina')
实时行情:   data = quotation.real(['600519', '000001'])
全市场代码:  codes = easyquotation.get_stock_codes()
```

### 注意事项

- 新浪行情接口是 HTTP 轮询，不是 WebSocket 推送，有 3-5 秒延迟
- 盘后（15:00 之后）数据不更新，返回的是收盘时的快照
- 批量查询全市场约 200ms 完成
- easyquotation 的 volume 和 turnover 字段含义与常规相反（volume 实际是成交额，turnover 是成交量），SinaGateway 中已做处理

## 三、akshare

### 基本信息

- **GitHub**: https://github.com/akfamily/akshare
- **安装**: `pip install akshare`
- **账号**: 无需注册
- **平台**: Mac / Windows / Linux
- **底层数据源**: 东方财富、同花顺等网站的公开接口

### 能获取的数据

| 数据类型 | 接口 | 说明 | 稳定性 |
|---|---|---|---|
| **涨停池** | `stock_zt_pool_em(date)` | 涨停股票列表，含连板数、所属行业、封板资金、封板时间 | 稳定，但只有近1个月数据 |
| 概念板块列表 | `stock_board_concept_name_ths()` | 同花顺概念板块名称和代码（375个） | 稳定 |
| 行业板块列表 | `stock_board_industry_name_ths()` | 同花顺行业板块（90个） | 稳定 |
| 个股基本信息 | `stock_individual_info_em(symbol)` | 总股本、流通股、总市值、行业、上市时间 | 较稳定，偶尔限流 |
| 概念成分股 | `stock_board_concept_cons_em(symbol)` | 某概念板块的所有成分股 | **不稳定，频繁被限流** |
| 行业成分股 | `stock_board_industry_cons_em(symbol)` | 某行业板块的所有成分股 | **不稳定，频繁被限流** |

### 不能获取的数据（或不可靠）

- 概念成分股查询被东方财富反爬，经常返回 `RemoteDisconnected`
- 涨停池历史数据只保留近 1 个月，早期数据返回空
- 无实时行情推送能力
- 无交易能力

### 限流问题

akshare 底层调用东方财富和同花顺的公开 Web 接口，存在反爬限制：

| 接口类型 | 限流情况 | 解决方式 |
|---|---|---|
| 涨停池 `stock_zt_pool_em` | 偶尔限流 | 请求间隔 0.3 秒 |
| 同花顺概念/行业列表 | 基本稳定 | 正常使用 |
| 东方财富概念成分股 | **频繁被断开** | 无法可靠使用 |
| 个股信息 `stock_individual_info_em` | 偶尔限流 | 加缓存，减少请求 |

Mac 本地和阿里云 Windows 服务器上都遇到了东方财富接口的限流问题，不是网络环境问题，而是东方财富服务端的反爬策略。

### 本项目中的使用方式

**1. 涨停池数据（scan_high_board.py V1）**

最初的扫描脚本用 `stock_zt_pool_em` 获取每天的涨停池和连板数。但发现只有近 1 个月数据，导致一年内只找到 2 次 6 晋 7。后来改用 baostock 全市场日线直接计算连板。

**2. 个股流通股本**

用 `stock_individual_info_em` 获取流通股本，用于计算竞价换手率。因为限流问题，后来改为用 baostock 日线的 `成交量/换手率` 反算。

**3. 行业板块验证**

用同花顺的 `stock_board_industry_name_ths()` 获取行业板块列表，辅助验证 baostock 的行业分类。

### 使用示例

```
涨停池:     ak.stock_zt_pool_em(date='20260410')
概念列表:   ak.stock_board_concept_name_ths()
个股信息:   ak.stock_individual_info_em(symbol='600519')
```

### 注意事项

- 东方财富的 `_em` 后缀接口频繁被限流，生产环境不可靠
- 同花顺的 `_ths` 后缀接口相对稳定
- 涨停池数据只有近期，不适合做长周期回溯
- 建议加 `time.sleep(0.3~1)` 控制请求频率

## 四、easytrader（实盘交易）

### 基本信息

- **GitHub**: https://github.com/shidenggui/easytrader （9.6k Star）
- **安装**: `pip install easytrader`
- **平台**: 仅 Windows（需要操控 GUI 窗口或连接 miniQMT）
- **费用**: 免费

### 支持的交易模式

| 模式 | 原理 | 需要什么 | 稳定性 |
|---|---|---|---|
| **同花顺模式** | pywinauto 模拟操作 xiadan.exe GUI 窗口 | 同花顺客户端 + 任意券商账号 | 中（客户端更新可能失效） |
| **miniQMT 模式** | 调用 xtquant 官方 API | miniQMT 客户端 + 支持的券商账号 | 高（官方接口） |
| **雪球模式** | HTTP 调用雪球组合 API | 雪球账号 | 低（仅模拟组合） |

### 能执行的操作

| 操作 | 接口 | 说明 |
|---|---|---|
| 买入 | `trader.buy(code, price, amount)` | 限价买入 |
| 卖出 | `trader.sell(code, price, amount)` | 限价卖出 |
| 撤单 | `trader.cancel_entrust(entrust_no)` | 按委托号撤单 |
| 查资金 | `trader.balance` | 总资产、可用金额、冻结金额、股票市值 |
| 查持仓 | `trader.position` | 代码、数量、成本价、市值、盈亏 |
| 查当日委托 | `trader.today_entrusts` | 委托列表 |
| 查当日成交 | `trader.today_trades` | 成交列表 |

### 本项目中的使用方式

**交易网关服务（trade_server.py）**

运行在 Windows 云服务器上，通过 Flask 暴露 REST API，Mac 端通过 HTTP 调用。

工作流程：
1. 同花顺 xiadan.exe 在远程桌面会话中登录平安证券
2. trade_server.py 通过 easytrader 连接到 xiadan.exe
3. Mac 端发送 HTTP 请求到 `http://云服务器IP:8888/api/order`
4. trade_server.py 接收请求 -> 风控检查 -> 调用 easytrader 下单
5. 返回委托结果给 Mac

配置文件 `config.yaml` 中指定券商类型和连接参数。

### 前置条件

**同花顺模式：**
- Windows 机器上安装同花顺客户端
- 用券商账号登录 xiadan.exe 的委托窗口
- xiadan.exe 窗口必须保持打开（不能最小化到托盘）
- easytrader 需要在有桌面会话的环境中运行（SSH 启动不行，需要远程桌面或本地桌面）
- 需要安装 `pywin32` 依赖

**miniQMT 模式：**
- 在支持的券商开户（国金、中泰、华鑫等）
- 申请 miniQMT 权限
- 下载安装 miniQMT 客户端
- miniQMT 客户端以"极简模式"登录并保持运行

### 注意事项

- 同花顺客户端更新后可能导致 easytrader 的 GUI 定位失效，需要等 easytrader 更新适配
- 同花顺模式必须在远程桌面会话中运行，不能通过 SSH 启动
- 下单速度：同花顺模式约 1-2 秒/单，miniQMT 模式约 100ms/单
- 平安证券已验证可用，国金证券 miniQMT 待开通

## 五、云服务器交易网关（自建 API）

### 基本信息

- **服务器**: 阿里云 ECS `123.57.94.129`
- **操作系统**: Windows Server
- **服务端程序**: `C:\trade_server\trade_server.py`（Flask）
- **客户端程序**: Mac 本地 `trade_client.py`
- **端口**: 8888
- **鉴权**: HTTP Header `Authorization: Bearer <secret_key>`

### 架构

```
Mac 本地                              阿里云 Windows 服务器
trade_client.py                       trade_server.py (Flask:8888)
      │                                     │
      │  HTTP POST/GET                      ├── 鉴权校验（密钥）
      │  ───────────────────────────→       ├── 风控检查（5层）
      │                                     ├── easytrader
      │  ←───────────────────────────       │     └── 同花顺 xiadan.exe
      │  JSON 响应                           │         └── 平安证券
      │                                     │
```

### 风控规则

下单请求在到达 easytrader 之前，依次通过 5 层检查:

| 层级 | 检查内容 | 当前配置 |
|---|---|---|
| 1 | 交易时段 | 只允许 9:30-15:00 下单 |
| 2 | 黑名单 | 拒绝 ST、*ST 股票 |
| 3 | 单笔金额 | 不超过 50,000 元 |
| 4 | 每日累计 | 不超过 200,000 元 |
| 5 | 仓位比例 | （待实现） |

风控配置在 `C:\trade_server\config.yaml` 中修改。

### 已知问题和注意事项

1. **远程桌面断开后不要注销**: 关闭远程桌面时选"断开连接"而不是注销，否则同花顺窗口会关闭，easytrader 连接失效
2. **窗口句柄失效**: 如果服务报 `Handle xxx is not a valid window handle`，说明同花顺窗口状态变了，需要在远程桌面中重启 start.bat
3. **32位应用警告**: 启动时会显示 `UserWarning: 32-bit application should be automated using 32-bit Python`，这是因为同花顺 xiadan.exe 是 32 位程序，不影响功能
4. **查询响应时间**: 同花顺 GUI 操作模式下，查询接口响应约 1-3 秒（easytrader 需要操控窗口切换页面读取数据），建议客户端 timeout 设置 30 秒
5. **下单风控**: 交易时段外（9:30 之前、15:00 之后）下单会被风控拦截，返回"非交易时段"
